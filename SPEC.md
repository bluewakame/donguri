# どんぐりアプリ 仕様書

---

## 0. このアプリの思想

**「デジタルの報酬が、リアルな行動を変えられるか」**

どんぐりアプリは、ゲームである前に **社会実験** である。

### 問い

- 🌰 どんぐり（デジタル報酬）があれば、人は今まで行かなかった場所へ歩くか？
- 🏪 ゲームの動機があれば、知らない店舗に入るか？
- 🌳 個人の小さな行動の積み重ねが、街の空気を変えるか？

### 仮説

> ゲームメカニクス × 位置情報 × 小さな報酬 = リアルな街の活性化

人は「楽しさ」があれば動く。
その楽しさを設計することで、街歩き・店舗訪問・地域探索という
**現実の行動を、意図的に引き出せる** のではないか。

### 検証したいこと

| レベル | 問い |
|--------|------|
| 個人 | デジタル報酬で、歩く距離・行動範囲が変わるか |
| 個人 | ゲームの目的があれば、未知の店舗に入る心理的ハードルが下がるか |
| 社会 | それが集積したとき、街の回遊率・店舗への流入が変化するか |
| 社会 | ゲームが「街との関係性」を変えるか |

---

## 1. アプリ概要

どんぐりアプリは、現実世界を歩きながら **「どんぐり」を集めて森を育てる位置情報ゲーム** である。

ユーザーは地図上に表示される「葉っぱ」や、店舗に設置されたQRコードを利用してどんぐりを集め、自分の森を成長させる。

本アプリは、**街歩き・探索・店舗訪問を促す社会実験的ゲーム** として設計されている。

---

## 2. ゲームコンセプト

ユーザーは **「冒険者」** として街を歩き、

- 🌿 道路に落ちている **葉っぱを拾う**
- 🏪 店舗を訪れて **どんぐりを入手する**

ことでどんぐりを集める。

集めたどんぐりによって **「森」** が成長する。

---

## 3. 基本システム

### 3.1 通貨

#### どんぐり 🌰

ゲームの基本通貨。

| 項目 | 内容 |
|------|------|
| 用途 | 森を育てる、将来的に店舗ポイントとして利用 |
| 入手方法 | QRコード / 葉っぱ交換 / ランダム取得 |

#### 金どんぐり ✨🌰

レアアイテム。

| 項目 | 内容 |
|------|------|
| 取得確率 | 5% |
| 用途 | 将来の拡張機能で使用予定 |

#### 葉っぱ 🌿

探索で入手できる素材。

| 項目 | 内容 |
|------|------|
| 出現場所 | 地図上にランダム生成 |
| 交換レート | 🌿 5枚 → 🌰 1個 |

---

## 4. 地図システム

アプリは地図ライブラリ（Leaflet）を使用し、現在地を取得する。

地図には次のオブジェクトが表示される。

| オブジェクト | 説明 |
|-------------|------|
| プレイヤー | 現在位置を示す |
| 葉っぱ 🌿 | プレイヤー周辺にランダム生成。クリックすると取得できる |
| 店舗マーカー 🌰 | 店舗の位置を示す。QRコードが設置されている |

---

## 5. QRコードシステム

店舗にはQRコードが設置されている。

| 項目 | 内容 |
|------|------|
| QRコード形式 | `donguri:店舗ID` |
| 報酬 | 🌰 +1 |
| 制限 | 1店舗 1日 1回 |

### 社会実験との接続点

QRコードの存在は、ユーザーに **「店舗に物理的に入る」** という行動を要求する。
これはゲームの文脈において、**「知らない店舗への入店ハードル」を意図的に下げるための設計** である。

---

## 6. 森システム

どんぐりの数に応じて森が成長する。

### 成長ルール

🌰 10個 → 🌳 1本

### 森レベル

| 本数 | レベル |
|------|--------|
| 0本 | まだ森はない |
| 1〜3本 | 小さな森 |
| 4〜7本 | 若い森 |
| 8本以上 | 深い森 |

---

## 7. 時間経過システム

どんぐりは時間経過で減少する。

| 項目 | 内容 |
|------|------|
| 減少原因 | 🐛 虫に食べられる |
| 仕様 | 一定時間ごとにどんぐり減少 |

### 設計意図

時間経過による減少は、**継続的な探索行動を促すための仕組み** である。
「放置すると森が減る」という緊張感が、定期的な街への外出を動機づける。

---

## 8. 探索システム

プレイヤーが移動すると新しい葉っぱが出現する。

| 条件 | 結果 |
|------|------|
| 一定距離の移動 | 新しい 🌿 が生成される |

### 設計意図

移動そのものに報酬を与えることで、**「歩くこと」自体をゲームプレイとして成立させる**。
目的なく歩くのではなく、「葉っぱを探して歩く」という意図的な行動へ転換する。

---

## 9. 社会実験としての設計思想（詳述）

### 9.1 個人レベルの行動変容

このゲームは、プレイヤーに以下の行動変容を促すよう設計されている。

```
普段行かない道を歩く
  → 葉っぱが出現する
  → どんぐりが増える
  → 森が育つ
```

```
知らない店舗の前に立つ
  → QRコードが目に入る
  → 「1回だけスキャンしよう」という動機が生まれる
  → 店舗に入る
```

**「ゲームのため」という免罪符が、行動のハードルを下げる。**

### 9.2 社会レベルへの集積

個人の行動変容が多数集まったとき、何が起きるか。

- 普段は人が来ない時間帯・場所に人が回遊する
- 小規模店舗に「ゲーム経由」の来客が生まれる
- 地域に対する「認知範囲」がプレイヤーの中で広がる

これは、**広告でも補助金でもなく、ゲームが街を動かす可能性** の検証である。

### 9.3 実験としての誠実さ

このアプリは「楽しいゲーム」として提供されるが、
その背後には「これで本当に街が変わるのか」という問いへの真剣な関心がある。

ゲームが機能するかどうかは、**プレイヤーの行動が答える**。

どんぐりアプリは

```
探索 × 街 × ゲーム × 店舗 × 社会実験
```

を組み合わせた位置情報ゲームである。

ユーザーは街を歩いてどんぐりを集め、自分だけの森を育てていく。
